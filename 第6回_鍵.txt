【秘密鍵と公開鍵】

鍵の役割
　・ログイン認証(一番一般的)



コマンド　sshkey-gen がだめだったので

PuTTYを使う
(C:\Program Files (x86)\WinSCP\PuTTYのputtygen.exe)
ユーザー名とパスでもログインできるけど、
もう一個の手法が鍵
sshの設定をサーバー側ですると鍵認証が出来るようになる
~/.ssh/anthraiation_keysに公開鍵をはる
ユーザーは秘密鍵を持っておいて、ログインするときに
ユーザー名・鍵でログインできる
(パスワード要らない)

秘密鍵から公開鍵は出来る
公開鍵はみんなに見られていい
公開鍵から秘密鍵には戻せないから(非可逆)

秘密鍵は自分のパソコンの中で閉じとくようなもの
3人サーバーにもぐりたい人がいるなら、
3人それぞれが秘密鍵と公開鍵を持たないといけない

いまはサーバーがGitHub
SSH and GPG keys
→New SSH key

~/.ssh/configファイルを作成、以下を書く
Host github github.com
  HostName github.com
    IdentityFile ~/.ssh/office_private_key.pem <!--秘密鍵の場所を書く-->
      User git

上記参照。めちゃめちゃわかりやすい。

群さん追記
同じパソコンであれば、鍵は1つでよい
が
GitHubで同じ鍵を登録できないようになっているため
2つ鍵を作って運用するしかありません。

2つ鍵を作る場合は下記参照
https://qiita.com/yampy/items/24638156abd383e08758

また、鍵を作る際は
[url "github:"]
    InsteadOf = https://github.com/
    InsteadOf = git@github.com:
の意味をわかっていないといけない

｀git remote add｀のコマンドでリモートリポジトリを登録したと思います。
このときにプロトコルを指定できます。
GitHubでリポジトリを最初作った時にローカルで行うコマンドが書いてたと思います。
このでかかれているのが https プロトコルを使った書き方になってます。
なので河田さんもhttpsプロトコルを使うようになっています。
ただ鍵認証でpushするにはsshプロトコルかgitプロトコルを使う必要があったかと思います。
｀git remote set-url｀でsshプロトコルを使う方法に書き換えてもいいですが、めんどくさいので
｀.gitconfig｀で ｀InsteadOf｀ を書いておけば、`https://github.com/`
ときたら ｀git@github.com:｀ に置き換えてねという意味になります。
